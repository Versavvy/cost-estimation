Reused audio and text are not charged twice, so real costs are usually a little lower. Fine-tune the assumptions in the controls below and in Advanced assumptions.

1 Users & volume

One session is a single student taking one question through the full flow (upload → walkthrough → complete). Total monthly sessions = users × sessions per user.

Total sessions / month 1,000

2 How the question arrives

A single session uses just one input method. Pick the one to model — or flip to a mix if your students vary across the month.

Model a mix across sessions
Input mix is normalised to 100% in the maths.

3 Walkthrough depth

The build pipeline is 5 LLM calls (steps A · parallel B · demo C · 3A explanation · answer key D). A→B→C can regenerate when the self-check fails — the retry factor scales those three.

4 Interaction

After the demo the child works their own question: each typed attempt is validated by one LLM call, and each mid-step follow-up question (“ask”) is another.

5 Audio (Inworld TTS-2)

Charged per character. Audio narrates the 3A explanation, every solution & demo step, the intros / completion, and each piece of feedback / Q&A reply.

Defaults verified against live published pricing (June 2026): o3 & gpt-4.1 both $2 input / $8 output per 1M tokens; Gemini 3.1 Flash Image (Nano-Banana-2) $0.067 per 1K image on the standard tier ($0.034 batch); Inworld TTS-2 $25 per 1M characters on demand — falling to $15 (at $300/mo), $12.50 (at $1,500/mo), and $10 at enterprise scale. omni-moderation is free. Cached input on the OpenAI models is ~$0.50/1M; the Batch API is ~50% off. If you have negotiated or volume rates, type them in — these are list prices, not your invoice.
